Instant Restaurant
Offline Eco-System

A fully offline restaurant ecosystem — five interconnected apps communicating without internet, powered by a local server running inside Flutter and IoT hardware.

Platforms POS, KDS, Kiosk, Token Display, Captain App
Architecture 100% Offline
Role System Architect & Product Manager
01 — The Problem

Internet dependency kills restaurant operations

Most restaurant POS and management systems require a constant internet connection. In many regions — and especially during peak hours — network outages cause order chaos, payment failures, and frustrated customers. A restaurant can't pause service because the WiFi went down.

Existing "offline" solutions either lose data, fail to sync, or degrade into limited modes that miss critical functionality. No solution offered a complete, multi-device ecosystem that genuinely works with zero internet.

02 — The Solution

A local server inside Flutter itself

The breakthrough was building a Dart-based local HTTP server that runs inside the Flutter application itself. This server handles all data, routing, and communication between the five ecosystem apps — without any cloud dependency.

All five apps (POS, Kitchen Display System, Self-Service Kiosk, Token Display, and Captain App) discover the local server on the network and communicate via REST APIs over LAN. IoT devices (ESP32 modules) handle physical triggers like buzzers and status lights.

03 — Key Features

Five apps, one ecosystem, zero internet

POS Terminal

Full point-of-sale with menu management, billing, split payments, discounts, and report generation — runs as the primary hub and local server.

Kitchen Display System (KDS)

Real-time order display for kitchen staff. Orders appear instantly when placed, with status updates flowing back to POS and Token Display.

Self-Service Kiosk

Customer-facing ordering interface. Guests browse the menu, customize their order, and place it directly — reducing wait time and staff dependency.

Token Display

Large-screen display showing order token numbers and their status. Customers know exactly when their order is ready without crowding the counter.

Captain App

Waiter/captain app for tableside ordering. Orders go directly to KDS without passing through the POS — faster service for dine-in.

IoT Integration (ESP32)

Physical hardware integration — buzzers for order ready alerts, LED indicators for kitchen status, all communicating over the local network.

04 — System & Architecture

The most complex system I've built

The POS app runs a Dart HTTP server that acts as the system's brain. All other apps connect to this server over the local WiFi network. Service discovery is automatic — when a KDS or Kiosk app launches, it finds the POS server without manual configuration.

Data is persisted in PostgreSQL on the local machine for production-grade reliability. The Node.js layer handles cloud sync when internet is available — pushing daily reports, analytics, and backups without interrupting live operations.

ESP32 microcontrollers (programmed in C) communicate with the Dart server via HTTP, triggering physical outputs like order-ready buzzers and kitchen status indicators. The entire system can run for months without touching the internet.
